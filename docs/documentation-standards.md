# Documentation Standards

## Document Metadata

Every documentation file inside `docs/` must begin with:

> Version: x.y.z
>
> Status: Pending | Draft | Approved | Deprecated
>
> Last Updated: YYYY-MM-DD
>
> Owner: Project Team
>
> Category: Architecture | UI/UX | Database | Backend | Frontend | API | Deployment | Testing | Requirements | ADR

---

## Status Definitions

Pending
The document exists but has not been written or reviewed.

Draft
Content exists but is still under review.

Approved
Reviewed and considered the current source of truth.

Deprecated
No longer authoritative. Kept only for historical reference.
